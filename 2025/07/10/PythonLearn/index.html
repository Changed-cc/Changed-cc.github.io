6.20：正则表达式（Regex）

一、基础知识

基本顺序：匹配单元（字符/分组/集合） + 限定符（数量、出现次数）

常见字符：

  • .：匹配除换行符之外的任意单个字符。

  • \d：匹配数字。

  • \w：匹配字母、数字、下划线。

  • \s：匹配空白。

  • *：匹配0次或多次。

  • +：匹配1次或多次。

  • ?：匹配0次或1次。

  • {n}：匹配确定n次。

  • []：匹配方括号内的任意一个。

  • ()：捕获分组。

  • ^、$：匹配行首、行尾。

  • |：匹配多个选项之一。

    Python中的常用方法

    方法 意义
    re.match() 在字符串开头匹配
    re.search() 搜索字符串中第一个匹配
    re.findall() 找到所有匹配，返回列表
    re.finditer() 找到所有匹配，返回迭代器
    re.sub() 执行字符串替换
    re.split() 按照匹配进行分割
    re.compile() 预编译正则对象以复用

·任意一，\d数字，\w字母

*零或多，+一或多，?零或一

{n}, {n,}, {n,m} 控制次数

^行首，$行尾

()分组，[]匹配范围

写正则表达式的思路：regex101工具

1.先分块，确定结构和边界，比如是什么类型，有几位

2.对分块的每一组去写正则表达式，再拼起来

3.编写时

        先写整体位数：\d{18}

        再逐段替换，每次改一小段用re.findall或re.search

一、HTTP工作原理与网页结构

HTTP协议：是一种超文本传输协议，是客户端和服务端通信的规则，本质是客户端发出请求（Request），服务器返回响应（Response）

请求

URL：统一资源定位符

方法 用途 爬虫中常用吗？
GET 获取资源 常用（网页、图片、API数据）
POST 提交数据 常用（表单提交、登录）
PUT/DELETE 更新、删除资源 爬虫少用

Header：请求头，是客户端传给服务器的附加信息，用于说明客户端的用户身份

Header 用途
User-Agent 告诉服务器你是哪个浏览器（防爬关键，建议伪装成浏览器）
Referer 告诉服务器你是从哪个页面跳转过来的（有时验证来源）
Cookie 携带身份凭证、会话信息
Accept 表示可接受的返回类型，例如 text/html、application/json

二、机器学习基本概念

理解：用训练集的特征和标签教机器建好模型，模型用损失函数 + 优化器学规律，避免过拟合或欠拟合，用交叉验证和评估指标去考察它的表现。

scikit-learn中文社区

SciPy 模块列表 | 菜鸟教程

训练集/测试集/验证集

验证集：用于模型调参和选择最优模型结构的数据集，不参与模型参数学习。 （机器用它调参数）

测试集：用于评估模型泛化能力和最终性能的数据集，模型在训练过程中未接触过测试集。（检查用它检查机器学得怎么样）

特征、标签、样本

特征（Feature）：用于描述数据对象的可观测属性或变量。
标签（Label / Target）：数据对象的目标值或分类标识，用于监督学习任务。 （即要学的结果） 样本（Sample）：数据集中独立的一条记录，由特征和标签组成（监督学习场景）。

模型、损失函数、优化器

模型（Model）是机器学习算法在训练数据基础上学到的映射关系或决策函数，用于对未知数据进行预测。（就是机器学到的规则，例如一条直线、一个决策树）
损失函数（Loss Function）用于度量模型预测输出与真实标签之间的误差，是模型训练时优化的目标。（就是机器的扣分公式，算学错了多少）
优化器（Optimizer）是用于调整模型参数以最小化损失函数的算法或方法，例如随机梯度下降（SGD）、Adam。（即机器“改正答案”的策略，比如每次改一点点）

过拟合、欠拟合

过拟合（Overfitting）是指模型在训练集上表现很好，但在新数据或测试集上性能下降的现象，通常是由于模型过于复杂，记住了训练数据的噪声或细节。
    ——即硬背训练集，偶遇新数据表现差
欠拟合（Underfitting）是指模型在训练集上和测试集上均表现不佳，未能充分学习数据的内在规律，通常是由于模型过于简单或训练不足。
    ——即常见的，训练数据都学不好

交叉验证

交叉验证（Cross Validation）是一种模型评估方法，通过将数据集分成若干子集，轮流用部分数据训练、剩余数据验证，综合各次验证结果评估模型性能，常用的如 K 折交叉验证。
    ——把数据分成多份，轮流训练和测试，确保机器学得稳，不依赖某一份数据

评估指标

准确率（Accuracy）：正确预测样本数与总样本数之比。
    ——对的总数/全部数（适合数据均衡时）

精确率（Precision）：预测为正例中真实为正例的比例。
    ——重视结果准确

召回率（Recall）：真实正例中被正确预测为正例的比例。
    ——重视不漏掉

F1 分数（F1-score）：精确率和召回率的调和平均数，用于权衡二者。

均方误差（MSE）：预测值与真实值误差平方的平均值，用于回归任务。

AUC（Area Under Curve）：ROC 曲线下的面积，用于衡量模型对正负样本的区分能力。
    ——模型区分正负样本的能力，即分类好坏的整体表现

三、机器学习常见的算法原理

（一）有监督学习（回归与分类）

1. 线性回归

    线性回归是一种用于建模因变量（目标值）与一个或多个自变量（特征）之间线性关系的监督学习算法，通过最小化预测值与真实值的均方误差来拟合最优线性模型。
    ——机器画一条直线（或平面、高维超平面），让它尽量贴近训练数据的点，用这条线预测新数据的结果。

误差函数：

$$
y_i = w \cdot x_i + b + \varepsilon_i
$$

假设误差$\varepsilon_i$是独立同分布，并符合均值为0，方差为$σ$的高斯分布，独立就是两个变量之间没有关系，同分布就是符合同一个分布规律，高斯分布也就是正态分布。可以将误差函数带入正态分布函数求得似然函数,似然函数采用极大似然估计可以得到损失函数。

$$
L(w, b, \sigma^2) = \prod_{i=1}^{n} \frac{1}{\sqrt{2 \pi \sigma^2}}
\exp \left( -\frac{(y_i - (w x_i + b))^2}{2 \sigma^2} \right)
$$

梯度下降： 想象你站在山坡上（损失函数的曲面上），每次都往坡最陡的方向走一小步，目标是走到山谷（最小损失）。

正则化： 正则化就是给模型加“约束”，防止它学得太复杂、记住训练集的噪声（过拟合）。可以想象：正则化像在模型上加了一根“皮筋”，防止它过度弯曲贴合数据。实现方法就是在损失函数中加入惩罚项

2.逻辑回归

    逻辑回归是一种用于二分类或多分类的监督学习算法，通过对特征进行线性组合后，使用 sigmoid 或 softmax 函数将输出映射为概率，从而实现分类。
    ——逻辑回归不是回归，而是分类，它通过画一条线将数据分为两类，输出结果为判断这个东西属于某一类的概率。

3.KNN（K近邻）

    K 近邻算法是一种基于实例的监督学习方法，通过测量输入样本与训练样本的距离，选择最近的 K 个邻居，并根据邻居的类别多数决定预测结果。
    ——遇到新数据时，看看它周围最近的K个邻居是什么类别，投票决定它的类别

算法三要素：

  • K值的选取

  • 距离度量的方式

  • 分类决策规则

4.决策树

    决策树是一种基于特征条件递归划分数据空间、形成树状结构的监督学习算法。
        —— 提出多个问题，回答是或不是，最后落到叶子节点给出预测结果

算法：

  • ID3算法：以信息增益度量属性选择，选择分裂后信息增益最大的属性进行分裂，该算法处理的是离散型的数据（如果数据是连续型的，也要转为离散型）

信息熵是用来衡量一个随机变量出现的期望值，如果信息越不确定，熵值越大，如果一件事一定发生，即概率为1，则其熵值为0

  • C4.5算法：基于ID3算法进行改进和提升，可以处理连续型的数据

    改进点有信息增益率、处理连续值、分裂点、缺失值的处理、过拟合问题

  • CART（Classification and Regression Tree）：分类回归树

    

   随机森林是由多个决策树构成的集成学习模型，通过多棵树的投票或平均提高预测准确性和鲁棒性（Robust）
        ——森林就是造很多决策树，让它们投票决定最终答案，这样更稳健

5.支持向量机（SVM——Support Vector Machine）

    支持向量机是一种监督学习算法，通过在特征空间中寻找最大化间隔的超平面，实现对数据的分类或回归。
    ——感觉上比线性回归更高级，因为它也是用一条线分出不同类别，而且可以使线两边的数据点尽可能分的更远

6.朴素贝叶斯

    朴素贝叶斯分类方法(Naïve Bayes)是利用贝叶斯定理来预测一个未知类别的样本属于各个类别的可能性，选择其中可能性最大的一个类别作为该样本的最终类别。

    ——就是已知先验概率，然后根据这个先验概率预测结果

“朴素”: 指一种假设，即特征条件独立假设，就是假设在同一个类别下，特征之间互不影响

7.集成算法

集成学习，顾名思义通过将多个单个学习器集成/组合在一起，使它们共同完成学习任务，以达到提高预测准确率的目的。有时也被称为“多分类器系统 (multi-classifier system)”。最终结果通常有以下几种决定策略：投票法、平均法、学习法

（1）Bagging
（2）随机森林
（3）Boosting
（4）AdaBoost （Adaptive Boosting，自适应提升）
（5）GBDT（Gradient Boosting Decision Tree，简称GBDT）梯度提升决策树
（6）XGBoost（GBDT的优化）
8.神经网络

    神经网络是一种由输入层、隐藏层和输出层构成的监督学习模型，基于由大量参数（权重、偏置）和非线性激活函数组成的神经元连接网络，用于逼近复杂函数映射。
    ——神经网络就是很多小函数（神经元）连成网，一层层提取数据特征，逐步拟合复杂关系。

6.23：无监督学习

（二）无监督学习

1.聚类（KMeans）

    KMeans 是一种无监督学习算法，通过迭代优化，将数据划分为 K 个聚类，使得簇内样本与簇中心的距离平方和最小。
    ——机器自己给数据分组（分为K个堆），每个组都有一个中心点，不需要事先指导答案，比如按照商场购买习惯给顾客分类。

2.降维（PCA）

    主成分分析（PCA）是一种降维方法，通过线性变换将高维数据投影到低维空间，最大化投影后数据的方差，从而保留数据的主要信息。
    ——数据原本的维度太高，机器帮你找几个重要的方向，把数据投影到这些方向上，减少维度同时尽量保留信息

五、深度学习和机器学习的区别

维度 机器学习 深度学习

参数化描述目标分类的错误（损失函数），通过优化器更新参数，使损失最小。损失函数就是在衡量“模型输出跟真实答案有多远”。

任务 损失函数 公式（简化）
回归 均方误差（MSE） $\frac{1}{n}\sum (y - \hat{y})^2$
分类 交叉熵损失 $- \sum y \log \hat{y}$

3.激活函数

作用：给神经网络带来非线性能力，让模型能拟合复杂模式。

函数 公式（简化） 特点
ReLU $f(x) = \max(0, x)$ 简单高效，主流选择
Sigmoid $f(x)=\frac{1}{1+e^{-x}}$ 输出(0,1)，易梯度消失
Tanh $f(x)=\frac{e^x - e^{-x}}{e^x + e^{-x}}$ 输出(-1,1)，居中但也可能梯度消失

4.正则化

防止过拟合，让模型泛化能力强。

方法 思路
L1/L2 正则化 惩罚大权重
Dropout 训练时随机丢弃部分神经元
数据增强 增加样本多样性（图像旋转、裁剪等）

5.优化器

优化器 特点
SGD（随机梯度下降） 简单，噪声大，收敛慢
Adam 自适应学习率，收敛快，主流选择
RMSprop 自适应，适合RNN

6.神经网络类型

（1）卷积神经网络CNN

  • 用于图像、视频。

  • 特征提取器：卷积层、池化层。

  • 可以自动提取局部空间特征（如边缘、纹理）。

🎯 关键词：图像、局部特征、共享权重。

（2）循环神经网络RNN

  • 处理序列数据（如文本、语音）。

  • 有“记忆”，输出依赖过去信息。

  • 常见变种：LSTM、GRU（解决长依赖问题）。

🎯 关键词：序列、记忆、上下文。

（3）生成对抗网络GAN

  • 包含生成器和判别器，互相对抗。

  • 生成逼真数据（图像、声音、视频等）。

  • 主要用于生成任务（图像生成、风格迁移）。

🎯 关键词：生成、对抗、假数据逼真。

7.深度学习工程师定位角色

  • 对模型架构：是用 CNN（图像）、RNN（序列）、Transformer（NLP）还是 GAN（生成任务）？ 每层的神经元数量、层数、连接方式。

  • 选对数据预处理方法：特征归一化？标准化？数据增强？ 如何划分训练 / 验证 / 测试集？

  • 选对损失函数和优化器：分类用交叉熵，回归用均方误差（MSE）。Adam、RMSProp、SGD 哪个更适合当前任务。

  • 调超参数：学习率多少？ Dropout 保留率设多少？ Batch size 设多少？

  • 防止过拟合，提升泛化能力：用正则化、早停、数据增强等手段提升模型泛化能力。

  • 诊断和改进模型：训练过程中监控损失、准确率曲线。 如果发现过拟合/欠拟合：该加数据？加层？换激活函数？

新手调优指南

  • 学习率：训练慢增大学习率，训练震荡或发散减小学习率

  • Batch Size：大模型或者性能好增大批次，小批次有助于泛化但训练时间更长

  • Dropout：太大可能欠拟合，太小可能防过拟合效果差

  • 优化器选择

  • 正则化方法

  • 模型监控：实时画出损失、准确率曲线

简单调优顺序建议

  1. 先调学习率和 batch size。

  2. 再调层数、神经元数、Dropout。

  3. 尝试不同激活函数、优化器。

  4. 用数据增强、正则化方法抗过拟合。

2.学习中不理解的地方的解释——计算机视觉与 PyTorch 基础笔记

torch.nn 模块

  • PyTorch 的神经网络模块工具箱。

  • 提供：神经网络层（Linear, Conv2d 等）、激活函数（ReLU, Sigmoid 等）、损失函数（MSELoss, CrossEntropyLoss 等）、容器（Sequential, ModuleList 等）。

  • 用于快速搭建神经网络，不需要手写公式。

torch.tensor

  • PyTorch 的基础数据结构，表示多维数组。

  • 类似于 NumPy 的 ndarray，但支持 GPU 运算和自动求导。

  • 用于存储输入、特征、权重、梯度等。

  • 可以是标量（0 维）、向量（1 维）、矩阵（2 维）或更高维度。

forward 函数

  • 每个继承自 nn.Module 的模型类必须定义 forward。

  • forward 决定数据的前向传播过程，即数据经过哪些层、如何组合。

  • 每次输入数据都会调用 forward 生成新的计算图。

backward 函数

  • 用于反向传播，计算损失对每个参数的梯度。

  • 调用 loss.backward() 后，每个参数张量会自动生成 .grad 属性，用于更新。

  • backward 实际上是基于动态计算图自动链式求导。

feature

  • 通常是网络中某层的输出，是张量。

  • 表示经过网络层后提取到的中间特征。

  • 例如卷积层输出的特征图。

特征本质上是张量，用于表示数据在某层的表达。

bias (偏置)

  • 网络层中的偏置项。

  • 在线性层中公式：output = input @ weight + bias

  • 作用：提升拟合能力，使模型不必经过原点。

  • bias 是可训练参数，有助于网络学习数据的平移。

梯度接近于 0 的含义

  • 参数的变化对损失影响很小。

  • 可能是：

    • 模型已收敛

    • 梯度消失（常见于深层网络或饱和激活函数）

梯度方向和大小决定了参数如何更新，梯度接近 0 意味着参数几乎不再更新。

激活函数中的负值

  • 激活函数会决定如何处理特征中的负值。

  • ReLU：负值直接置 0，可能导致神经元死亡。

  • Leaky ReLU, PReLU：负值保留较小斜率，防止神经元死亡。

  • Sigmoid, Tanh：负值可以输入，但在极值处可能导致梯度消失。

在设计网络时，负值本身不是问题，关键是激活函数的处理方式。

梯度的负值与特征负值区别

  • 特征负值：前向传播中某一层的输出值 < 0。

  • 梯度负值：反向传播中求出的梯度值 < 0，用于决定参数更新方向。

特征负值和梯度负值是两个不同阶段的概念，不要混淆。

6.26CNN模型

（一）CNN 是什么

卷积神经网络（Convolutional Neural Network，CNN）是一种深度学习模型，主要用于处理具有类似网格结构的数据，例如图像（像素网格）。

CNN 通过卷积层提取局部特征，以较少参数捕获空间关系，是计算机视觉中最成功的模型之一。

（二） CNN 的主要结构组成

CNN 通常包含以下几个基本层：

1. 卷积层（Convolution Layer）

  • 功能：使用卷积核（滤波器）滑动扫描输入数据，提取局部特征。

  • 参数：卷积核大小、步长（stride）、填充（padding）、通道数。

  • 输出：特征图（feature map），也称激活图。

公式（单通道简化版）：

Y(i,j)=m∑​n∑​X(i+m,j+n)⋅K(m,n)

其中 K 是卷积核。

2. 激活函数（Activation Function）

  • 常用：ReLU、Leaky ReLU、PReLU、Sigmoid、Tanh。

  • 功能：引入非线性能力。

  • ReLU 特点：将负值置 0，使计算简单、梯度传播稳定。

3. 池化层（Pooling Layer）

  • 常用：最大池化（Max Pooling）、平均池化（Average Pooling）。

  • 功能：降低空间尺寸、减小参数量、防止过拟合。

  • 原理：在局部窗口内取最大值或均值。

4. 全连接层（Fully Connected Layer, FC）

  • 功能：将卷积提取的特征映射到输出类别。

  • 结构：类似传统神经网络，每个神经元和前一层所有神经元相连。

5. 批归一化（Batch Normalization，BN）

  • 功能：加速收敛、减少梯度消失问题。

  • 原理：对每层的输出进行标准化。

6. Dropout（可选）

  • 功能：防止过拟合。

  • 原理：训练时随机丢弃一部分神经元。

（三）CNN 的工作流程

  1. 输入图像（张量形式，如 batch_size x channels x height x width）。

  2. 卷积层提取局部特征。

  3. 激活函数引入非线性。

  4. 池化层下采样特征。

  5. 多次卷积-池化组合提取高层特征。

  6. 特征展平（flatten）。

  7. 全连接层分类或回归输出。

  8. 计算损失，反向传播，更新权重。

（四）CNN 的主要优点

  • 局部感受野：只关注局部区域特征。

  • 权值共享：相同卷积核用于整个输入，参数少。

  • 平移不变性：对物体在图像中位置变化有较强鲁棒性。

（五）关键术语与概念

  • 卷积核（filter/kernel）：滑动窗口中的权重矩阵。

  • 步长（stride）：卷积核滑动的步幅。

  • 填充（padding）：输入边缘补零以控制输出尺寸。

  • 特征图（feature map）：卷积输出的结果。

  • 感受野（receptive field）：卷积神经元感知的原始输入区域。

  • 参数量：卷积层参数 = 卷积核尺寸 × 输入通道数 × 卷积核个数。

（六）常见经典 CNN 架构

网络 特点
LeNet 最早的 CNN，用于手写数字识别
AlexNet 引入 ReLU、Dropout，大幅提升 ImageNet 性能
VGG 堆叠小卷积核（3x3），结构简单效果好
GoogLeNet 引入 Inception 模块，多尺度特征融合
ResNet 引入残差连接，解决深层网络退化问题
DenseNet 特征图密集连接，提升信息流通

（七） CNN 的训练与推理流程

  1. 前向传播：输入数据流经卷积层、池化层、全连接层，计算输出。

  2. 计算损失：使用交叉熵等损失函数计算预测与真实的误差。

  3. 反向传播：计算各层参数的梯度。

  4. 优化器更新参数：SGD, Adam 等优化器根据梯度更新权重。

  5. 推理（eval 模式）：禁用 Dropout、BN 的训练行为，进行预测。

（八）初学者应掌握的 CNN 重点

  1. 卷积、池化、全连接层的作用和参数量计算。

  2. 数据维度变化：输入图像经过各层后 shape 如何变化。

  3. 卷积核、步长、填充如何影响特征图大小。

  4. forward 流程：会写简单的 forward 函数。

  5. 使用预训练模型（如 ResNet）做分类/特征提取。

  6. 梯度流动与权重更新基本原理。

6.26压力测试

（一）压力测试定义：

压力测试是一种通过模拟极端条件（如高并发用户、大数据量、长时间运行等）来评估系统性能、稳定性和可靠性的测试方法。目的是发现系统在超负荷运行时的瓶颈、潜在故障或性能下降点。

常见应用场景：

  • 软件/网站：模拟大量用户同时访问，测试服务器响应时间和崩溃阈值。

  • 硬件：如CPU、内存满负载运行，检查散热和稳定性。

  • 金融：评估银行或投资系统在市场剧烈波动时的承受能力。

工具举例：
JMeter、LoadRunner、Locust等。

（二）OpenVPN的作用

OpenVPN 是一个开源的虚拟专用网络（VPN）工具，用于创建加密的隧道连接，确保数据传输的安全性和隐私性。

主要用途：

  1. 安全远程访问：企业员工通过VPN安全连接内网资源（如文件服务器）。

  2. 隐私保护：加密网络流量，防止被窃听（尤其在公共Wi-Fi下）。

  3. 绕过地理限制：通过连接到其他地区的VPN服务器访问地域限制内容。

  4. 点对点安全通信：如跨地域办公室之间的加密数据传输。

特点：

  • 支持SSL/TLS加密，安全性高。

  • 可跨平台（Windows、Linux、macOS等）。

1.切换接入点（APN/Access Point）

定义：
接入点（APN）是设备连接蜂窝网络的网关配置，切换接入点指更改设备使用的网络连接路径，通常用于移动网络（如4G/5G）。

常见场景：

  1. 运营商切换：在不同运营商网络间切换（如漫游时）。

  2. 网络类型切换：从4G切换到专有APN（如企业内网、物联网卡）。

  3. 故障排除：当默认APN连接失败时，手动更换其他配置。

操作方法（以手机为例）：

  • Android/iOS：设置 → 移动网络 → APN → 选择或新增APN。

注意：错误配置可能导致无法上网，需谨慎操作。

2.VPN

VPN（Virtual Private Network，虚拟专用网络） 是一种通过公共网络（如互联网）建立加密隧道的技术，使得远程用户或设备能够像直接连接私有网络一样安全地访问资源。

作用：

结合计算机网络（如TCP/IP协议、路由、加密等），VPN 主要实现以下功能：

作用 计算机网络原理 实际应用
安全远程访问 使用 隧道协议（如IPSec、OpenVPN、WireGuard） 封装数据，在公共网络上建立私有通道 企业员工在家访问公司内网
数据加密 采用 SSL/TLS、AES 等加密算法，防止数据被窃听或篡改 在咖啡厅用公共Wi-Fi时保护隐私
绕过网络审查/地理限制 通过VPN服务器 伪装IP，使流量看起来来自其他地区 访问被封锁的网站（如某些国家的Google）
点对点安全连接 在互联网上模拟 局域网（LAN），使不同地点的设备直接通信 分公司之间安全传输数据

(1) 协议层（OSI/TCP/IP模型）

  • 应用层：SSL-VPN（如网页VPN）直接加密应用数据（如HTTPS）。

  • 传输层/网络层：IPSec VPN 在IP层加密，适用于整个网络流量。

  • 数据链路层：L2TP（二层隧道协议）可模拟局域网连接。

(2) 隧道技术（Tunneling）

VPN 通过 封装（Encapsulation） 将原始数据包（如HTTP请求）包裹在另一个协议（如IPSec或OpenVPN）中传输，避免被中间网络设备解析。

(3) 认证与加密

  • 认证：通过 用户名/密码、证书、双因素认证（2FA） 确保只有合法用户能连接。

  • 加密：使用 AES（高级加密标准）、RSA（非对称加密） 等算法防止数据泄露。

(4) NAT（网络地址转换）与VPN

  • VPN 可以绕过 NAT 限制，让内网设备（如摄像头）通过公网访问。

  • 企业VPN通常结合 NAT穿透 技术（如STUN/TURN）实现直连。

常见VPN协议对比

协议 加密方式 速度 适用场景
OpenVPN SSL/TLS + AES 中等 通用（平衡安全与速度）
IPSec AES/3DES + IKEv2 快 企业级VPN（iOS/Android支持）
WireGuard ChaCha20 + 轻量级 极快 移动设备、低延迟需求
L2TP/IPSec IPSec加密 较慢 旧设备兼容（逐渐淘汰）

（二）Transformer的介绍

传统序列模型（如RNN）一个词一个词顺序处理，而Transformer靠全局注意力，一次性看到全部词，并通过“注意力机制”来决定谁应该被关注。

设计 原因 具象化理解
自注意力 不需要记忆过去，直接计算词间相关性 拿一个矩阵就能表示词对词“谁看谁”
多头机制 提取不同角度的语义 一组头看词性，一组头看语义
并行训练 不像RNN串行，可以并行运算 GPU 一次计算整句话，无需等待前词处理完
位置编码 弥补没有序列感的缺陷 加上每个词的位置，使得“who is where”有意义
模型 核心思路 应用方向
BERT (2018) 双向编码器，MLM（掩码语言建模）+ NSP 情感分析、问答、分类
GPT系列 单向生成器，语言建模（预测下一个词） 对话生成、文本补全
T5 把所有任务统一为“文本到文本” 多任务通用模型
RoBERTa、ALBERT、DeBERTa等 在BERT上优化训练策略和结构 高精度NLP任务
LLMs（GPT-4、Gemini、Claude等） 数十亿参数，强化学习微调、提示学习等 万能型生成、推理、问答等任务

收获三：理解前后端两种不同的实现方式

Vue + SpringBoot 的前后端分离开发模式，现在转向 Python + FastAPI

方面 Vue + SpringBoot Vue + FastAPI
通信方式 RESTful API（JSON/XML） RESTful API（默认JSON，支持表单/文件）
后端框架 Java生态（强类型，注解驱动） Python生态（动态类型，装饰器路由）
前端集成 Vue通过axios调用SpringBoot接口 Vue同样通过axios调用FastAPI接口
跨域处理 Spring需加@CrossOrigin FastAPI用CORSMiddleware
开发速度 配置较复杂，但企业级稳定 快速原型开发，适合中小项目
性能 JVM优化后吞吐量高 Python解释型，但FastAPI异步性能优秀

关键：

  1. 理解RESTful API

  2. 区分前后端分别部署和混合部署

  • 混合部署方法：前后端代码在同一个库中；前端使用纯HTML/CSS/Js（或轻量框架jQuery）;或者还可以使用模板渲染（动态页面，使用jinja2等模板引擎）

目前我们项目应该使用的是混合部署，且使用jinja2模板引擎和FastAPI后端技术。
